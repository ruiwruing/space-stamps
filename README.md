# T.108《航天》邮票集邮展 · 项目交接文档

> 供下次继续设计时读取。最后更新：2026-07-21。

## 一、项目是什么

中国邮政 1986 年 T.108《航天》特种邮票（全套六枚，刘硕仁设计）的交互式线上展览。
流程：**黑白半调开场**（环形邮票转盘 + canvas 线框地球）→ 点击地球进入 → **彩色 3D 玻璃托盘轮播** → 点击邮票翻面看档案卡。
支持四种交互通道：触屏手势（AlloyFinger）、鼠标键盘、隔空手势（摄像头 MediaPipe）。

## 三、邮票数据与图源

- 数据在 JS 的 `STAMPS` 数组（6 项）：`no / title / subject / year / value / date / img(base64) / desc`
- 六枚：(6-1)乐声环宇·4分 / (6-2)天外归来·8分 / (6-3)雷震海天·8分 / (6-4)腾飞万里·10分 / (6-5)天地同音·20分 / (6-6)玉宇明灯·70分
- 图源（如需换高清图）：`https://stampwikistatic.11185.cn/media/stamp/532/headpic/{n}.png`，n=1 为全套总览，n=2~7 对应 6-1~6-6，均为 750×750 透明 PNG
- 图片以 **WebP base64** 内联（单枚约 25~38KB）。替换图片需重跑内联脚本（见 `/tmp/20260721_inline_assets.py` 思路：PIL 转 WebP→base64→字符串替换）

## 四、文件结构锚点（用注释标记定位，行号会漂移）

CSS 段：`BASE`（变量/星空/流星/轨道环/星图/航空边框/邮政元素）→ `STAGE`（玻璃托盘/轮播槽位/入场动画）→ `UI`（页眉/页码/提示条/盖戳/粒子/灯箱/隔空手势 UI）→ `开场`（半调/转盘/地球/加载条）→ `响应式`
JS 段：`STAMPS 数据` → `星空/星图生成` → `buildCards + revealCards` → `展台视差` → `轮播 goTo/next/prev` → `水印/盖戳/缩放/翻面/粒子` → `灯箱` → `AlloyFinger 绑定` → `鼠标/键盘兜底` → `隔空手势（AIR_ 开头）` → `initIntro（转盘+地球+退场）`

## 五、关键技术参数（调手感时改这里）

隔空手势灵敏度：
- `AIR_SWIPE_DX=.05`（位移阈值）、`AIR_SWIPE_V=.0008`（速度阈值）、`AIR_COOLDOWN=240`ms、`AIR_REARM_DX=.03`（回手装填距离）
- One-Euro 滤波（`minCutoff=1.2, beta=0.012`）：快动零延迟、慢动平滑
- 手势防抖 4帧/2次；遮挡宽限 700ms；摄像头 720p@60fps，`modelComplexity:0`（lite 模型低延迟）
- 手势语义：`point`=滑动翻页 / `pinch`=缩放开关 / `peace`=翻面 / `open`=待命
- **回手装填机制**：触发一次后必须回手到起点附近才重新装填，防止回程被误判为反向滑动（用户痛点"滑回最左边"的解法）

卡片尺寸：`--card-w:min(460px,86vw)`、`--card-h:card-w*1.18`、邮票 `max-width:97%`（用户要求：邮票大、边框小）

## 七、⚠️ 踩坑记录（必读）

1. **CDN 版本号绝不能凭印象写**：MediaPipe 只有 `hands@0.4.1675469240` 和 `camera_utils@0.3.1675466862` 可用（已验证 200），其他版本号 404。改 CDN 前必须 `curl -o /dev/null -w "%{http_code}"` 验证。
2. **预览环境不解析子目录相对路径**：`stamps-t108/xx.png` 这类引用会碎图 → 所以图片全部 base64 内联，页面零外部依赖。
3. **浏览器缓存**：用户多次看到旧画面 → 已加 no-cache meta；交付时提醒强制刷新。
4. **编辑含 base64 的大文件**：避开超长行；批量改动用 python 替换脚本 + **命中计数校验**（count!=1 即中止），比逐处 EditFile 安全。
5. **mediapipe-master.zip 是 Bazel 源码**，无预编译产物，不能直接用于浏览器。
6. 地球方案演进：播放三角 → 地图平移圆窗（球面感差）→ **canvas 球面投影线框地球**（经线椭圆收拢、背面线淡化、拖拽+惯性）。

## 八、用户审美偏好（从修改意见提炼）

- 邮票是主角：票要大、边框要窄，拒绝"框大票小"
- 喜欢黑白半调印刷质感的开场（参考图风格）
- 介绍信息不要常驻浮窗，收进背面、点击翻面才出现
- 地球要"真"：可旋转的球体模型，不是贴图
- 整体要求：有设计感、元素丰富、微交互到位
